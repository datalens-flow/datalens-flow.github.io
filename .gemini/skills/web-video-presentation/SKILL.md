# Web Video Presentation

把一篇文章或口播稿，一步步做成可录屏的"伪装成视频的网页"，可选合成口播音频。产出物 = Vite + React + TS 项目 + 按章节切分的音频。

## 适用场景

- "我有口播稿 / 一篇文章，帮我做成视频" —— 口播驱动的内容
- 想做 "动态 PPT"
- 16:9 横屏录屏，大字、留白、每屏都要有动效
- 教学 / 产品演示 / keynote 想要电影感
- B 站 / YouTube / 抖音视频内容

本 Skill **以方法论 + 协作流程为核心**。脚手架模板提供 token 和原语，但每个美学决策（配色、字型、动效气质）都应该针对你的主题重新设计 —— 不要照搬。

---

## 核心流程

### Step 1: 原始输入分析 & 产出物对齐
当收到一篇文章或想法时，**一次性**输出以下两样：
1. **口播稿 (Narration Script)**: 按节拍切分，结构为 `(chapter, step)`。
2. **Outline 规划**: 规定每个 step 的视觉呈现意图、画面节奏与信息密度。
   - **注意**: outline **只规划节奏**，不规划具体 CSS 属性或具体动画帧。

### Step 2: 停顿对齐 (Hard Checkpoints)
在开始编写代码前，与用户对齐以下 5 件事：
1. 口播稿节拍划分是否正确？
2. 视觉表现大纲 (Outline) 节奏是否满意？
3. 美学主题设计 (Theme Tokens - 字体, 色卡, 布局)？
4. 用到的外部静态素材 (图片/视频) 路径？
5. 音频合成机制的选择？

### Step 3: 开发阶段 (Coding)
- 基于 Vite + React + TS 结构。
- 逐个章节进行状态绑定，并使用 React-Flow 或 CSS Keyframes 制作高阶动效。
- 悬浮进度条设计，方便预览同时保证录制画面无杂质。

### Step 4: 音频合成 (Optional)
- 通过 CLI/API 合成口播音频并将其放置在 `public/audio/` 下。
